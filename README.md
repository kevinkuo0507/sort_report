排序演算法是電腦科學中的基礎概念，也是面試與競賽中的常見考題。本報告透過五種經典排序演算法的理論分析與互動式視覺化，深入探討不同排序方法的原理、效能特性與實務應用。

本報告包含以下內容：

## 氣泡排序法

- 原理：重複走訪陣列，比較相鄰元素，若順序錯誤就交換，較大的值會逐步「浮」到右側。
- 時間複雜度：
	- 最佳：$O(n)$（已排序且有提早結束判斷）
	- 平均：$O(n^2)$
	- 最差：$O(n^2)$
- 空間複雜度：$O(1)$
- 模擬頁面重點：
	- 顯示每輪比較與交換。
	- 提供比較次數、交換次數、輪次、耗時統計。
	- 支援速度調整、暫停/繼續、重置與步驟紀錄。

## 選擇排序法

- 原理：每一輪從未排序區間找出最小值，與目前起始位置交換。
- 時間複雜度：
	- 最佳：$O(n^2)$
	- 平均：$O(n^2)$
	- 最差：$O(n^2)$
- 空間複雜度：$O(1)$
- 模擬頁面重點：
	- 突出目前掃描位置與當前最小值更新。
	- 顯示每輪最終交換結果。
	- 提供互動控制與完整過程紀錄。

## 插入排序法

- 原理：將陣列分成已排序與未排序區間，逐步把 key 插入已排序區間正確位置。
- 時間複雜度：
	- 最佳：$O(n)$
	- 平均：$O(n^2)$
	- 最差：$O(n^2)$
- 空間複雜度：$O(1)$
- 模擬頁面重點：
	- 呈現 key 的提取、比較、位移與插入。
	- 可觀察近乎排序資料時的較佳表現。
	- 提供步驟紀錄與統計數據。

## 合併排序法

- 原理：使用分治法，先把陣列切成小區塊，再將已排序子陣列兩兩合併。
- 時間複雜度：
	- 最佳：$O(n \log n)$
	- 平均：$O(n \log n)$
	- 最差：$O(n \log n)$
- 空間複雜度：$O(n)$
- 模擬頁面重點：
	- 顯示合併區間、左右子陣列比較與寫回位置。
	- 以「合併輪次」呈現分治進度。
	- 統計比較次數、寫回次數與耗時。

## 快速排序法

- 原理：選定 pivot，將小於等於 pivot 與大於 pivot 的元素分割到兩側，再遞迴排序子區間。
- 時間複雜度：
	- 最佳：$O(n \log n)$
	- 平均：$O(n \log n)$
	- 最差：$O(n^2)$（pivot 選擇不佳時）
- 空間複雜度：
	- 平均：$O(\log n)$（遞迴堆疊）
	- 最差：$O(n)$
- 模擬頁面重點：
	- 強調 pivot、分割區間、比較與交換。
	- 顯示每次分割後 pivot 定位位置。
	- 提供比較次數、交換次數、分割次數與進度條。

## 綜合比較

| 演算法 | 最佳時間 | 平均時間 | 最差時間 | 空間複雜度 | 穩定性 |
|---|---|---|---|---|---|
| 氣泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 合併排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |
| 快速排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$~$O(n)$ | 不穩定 |

### 小結

- 小型或近乎排序資料：插入排序通常表現不錯。
- 需要穩定且一致的時間表現：合併排序較適合。
- 一般情況追求平均效率：快速排序通常較快，但需注意最差情況。

## 學習心得

### 視覺化對學習的助益
- 透過動畫化，我能更直觀理解「比較、交換、位移、分割、合併」等核心動作，而不是僅停留在數學公式的記憶。
- 暫停功能讓我能逐步觀察每一步的計算過程，對理解演算法邏輯有很大幫助；速度調整則方便觀察快速與慢速執行下的差異。
- 彩色編碼（比較中、交換中、已排序等）能快速定位目前正在進行的操作，降低認知負擔。

### 複雜度的實踐意義
- 同樣是 $O(n^2)$ 的排序法（氣泡、選擇、插入），在實際執行步驟與視覺行為上有明顯差異：
  - 氣泡排序幾乎每次都在交換，步驟最多。
  - 選擇排序每輪只做一次交換，但需要掃整個未排序區間。
  - 插入排序在近乎排序的資料上表現優異（接近 $O(n)$）。
- 分治法（合併排序、快速排序）在資料量增加時的效率優勢更加明顯；特別是當 $n > 100$ 時，$O(n^2)$ 的演算法明顯變慢。

### 穩定性的實際影響
- 選擇排序與快速排序的不穩定性，在某些應用中會導致結果差異，例如多鍵排序（先按主鍵穩定排序，再按副鍵）。
- 穩定排序更適合用於保持資料的原始相對順序，這在數據庫查詢結果排序中特別重要。

### 空間與時間的內部交換
- 合併排序為了保證 $O(n \log n)$ 的時間複雜度，需要額外的 $O(n)$ 空間儲存中間結果。
- 快速排序只需 $O(\log n)$ 的遞迴堆疊空間（平均情況），更節省內存，說明演算法設計中時間與空間的平衡取捨。

### 演算法選擇的實務考量
- 小型資料集（$n < 50$）：插入排序或選擇排序足夠快，實作簡單。
- 中型資料集（$50 \leq n \leq 10000$）：快速排序通常是最佳選擇，平均效率高。
- 大型資料集（$n > 10000$）：合併排序的穩定 $O(n \log n)$ 表現更可靠；若記憶體充足，也可考慮快速排序及其優化版本。
- 實務中的排序庫（如 Python 的 `sorted()` 或 `list.sort()`）通常採用 **Timsort**，結合區間分析與多種演算法的混合。

### 程式設計的核心概念
- **分治法的威力**：將大問題分解成小問題，再逐層整合，是大多數高效演算法的基礎。
- **遞迴 vs 疊代**：合併排序與快速排序相對容易用遞迴表達，但疊代版本對記憶體與棧空間的控制更精細。
- **邊界條件的重要性**：實作時務必小心處理空陣列、單元素、重複元素等邊界情況，這些往往是 bug 的根源。

### 特殊考量
- 快速排序選擇 pivot 的策略（首元素、末元素、隨機、中位數）會顯著影響效率，特別是對抗「最差情況」。
- 氣泡排序的「已排序化檢測」（`swapped` flag）是一個實用的提前終止最佳化，值得學習。
- 插入排序在「幾乎已排序」的資料上表現突出，適合用於增量排序或預排序列表的維護。
- 現實中的排序任務往往涉及複雜物件而非單純數字；穩定性、記憶體局部性、快取效應等因素都會影響實際性能。
