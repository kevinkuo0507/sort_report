## 氣泡排序法

- 原理：重複走訪陣列，比較相鄰元素，若順序錯誤就交換，較大的值會逐步「浮」到右側。
- 時間複雜度：
	- 最佳：$O(n)$（已排序且有提早結束判斷）
	- 平均：$O(n^2)$
	- 最差：$O(n^2)$
- 空間複雜度：$O(1)$
- 模擬頁面重點：
	- 顯示每輪比較與交換。
	- 提供比較次數、交換次數、輪次、耗時統計。
	- 支援速度調整、暫停/繼續、重置與步驟紀錄。

## 選擇排序法

- 原理：每一輪從未排序區間找出最小值，與目前起始位置交換。
- 時間複雜度：
	- 最佳：$O(n^2)$
	- 平均：$O(n^2)$
	- 最差：$O(n^2)$
- 空間複雜度：$O(1)$
- 模擬頁面重點：
	- 突出目前掃描位置與當前最小值更新。
	- 顯示每輪最終交換結果。
	- 提供互動控制與完整過程紀錄。

## 插入排序法

- 原理：將陣列分成已排序與未排序區間，逐步把 key 插入已排序區間正確位置。
- 時間複雜度：
	- 最佳：$O(n)$
	- 平均：$O(n^2)$
	- 最差：$O(n^2)$
- 空間複雜度：$O(1)$
- 模擬頁面重點：
	- 呈現 key 的提取、比較、位移與插入。
	- 可觀察近乎排序資料時的較佳表現。
	- 提供步驟紀錄與統計數據。

## 合併排序法

- 原理：使用分治法，先把陣列切成小區塊，再將已排序子陣列兩兩合併。
- 時間複雜度：
	- 最佳：$O(n \log n)$
	- 平均：$O(n \log n)$
	- 最差：$O(n \log n)$
- 空間複雜度：$O(n)$
- 模擬頁面重點：
	- 顯示合併區間、左右子陣列比較與寫回位置。
	- 以「合併輪次」呈現分治進度。
	- 統計比較次數、寫回次數與耗時。

## 快速排序法

- 原理：選定 pivot，將小於等於 pivot 與大於 pivot 的元素分割到兩側，再遞迴排序子區間。
- 時間複雜度：
	- 最佳：$O(n \log n)$
	- 平均：$O(n \log n)$
	- 最差：$O(n^2)$（pivot 選擇不佳時）
- 空間複雜度：
	- 平均：$O(\log n)$（遞迴堆疊）
	- 最差：$O(n)$
- 模擬頁面重點：
	- 強調 pivot、分割區間、比較與交換。
	- 顯示每次分割後 pivot 定位位置。
	- 提供比較次數、交換次數、分割次數與進度條。

## 綜合比較

| 演算法 | 最佳時間 | 平均時間 | 最差時間 | 空間複雜度 | 穩定性 |
|---|---|---|---|---|---|
| 氣泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 合併排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |
| 快速排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$~$O(n)$ | 不穩定 |

### 小結

- 小型或近乎排序資料：插入排序通常表現不錯。
- 需要穩定且一致的時間表現：合併排序較適合。
- 一般情況追求平均效率：快速排序通常較快，但需注意最差情況。

## 學習心得

- 透過動畫化，我能更直觀理解「比較、交換、位移、分割、合併」等核心動作。
- 同樣是 $O(n^2)$ 的排序法，在實際步驟與視覺行為上差異很大。
- 分治法（合併排序、快速排序）在資料量增加時，效率優勢更明顯。
- 加入暫停、速度控制與步驟紀錄後，更容易用於課堂展示與自我檢查。
